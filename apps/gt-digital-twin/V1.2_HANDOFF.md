## What changed in v1.2

### 1. Typography (responding to "headings too modern / numbers can be more professional")

- **Display font replaced.** v1.1 used Instrument Serif *italic* at clamp(2.4–3.6rem)
  for hero and every section title. That reads editorial / trendy. v1.2 swaps in
  **Source Serif 4** (upright, optical-sizing, semibold) — institutional, restrained,
  professional. No italic.
- **Body untouched.** Geist remains the body/UI face (the user said this was fine).
- **Numbers fully tabularized.** Every large numeric class (`num-xl`, `num-lg`,
  `num-md`) is now IBM Plex Mono with `tabular-nums lining-nums zero`. KPI tiles,
  ribbons, table cells, and chart axes all align on the same column grid. New
  utility class `.num-pro` exposes the same treatment for ad-hoc numerics.
- **`.display` and `.display-sm`** are the new heading utilities. The historical
  `font-serif italic text-3xl ...` pattern across eight components was replaced
  with `display text-3xl ...`.

### 4. Forecast chart correctness — which one was wrong, why

There are two related artifacts:

- **`forecast_trajectories.png` (v1.1)** — the static repo image rendered by
  `apps/gt-digital-twin/scripts/precompute.py`. It shows the 5-panel cumulative
  envelope (Net P&L, spark margin, LTSA owner cost, generation, fired hours)
  plus a how-to-read legend. This is the canonical run_forward output that
  ships with the repo today. It is **not** the 6-panel chart the user described.
- **The user-described 6-panel chart** (SEAS5 temp ensemble · forward LMP ·
  Henry Hub · generation · gross margin · Net P&L CDF) is the natural
  "diagnostic stack" expected of a forward-engine output but is **not currently
  emitted by the engine**. The forward run directory `forward_20260527_141023`
  referenced by the user does **not exist in the repo on disk**; we checked
  `data/outputs/lockport/forward_runs/`. So the screenshot the user is
  comparing against is either an off-repo artifact or a development branch
  that didn't land.

v1.2 resolution: build the diagnostic stack the user wants as an interactive
Recharts component on the dashboard side, sourced from the data we do have.
The endpoint values of the engine panels (generation, gross margin, net P&L)
are real `run_forward` per-path outputs. Monthly shape is a documented
canonical NYISO Zone F CCGT seasonality so the curves are continuous.
Temperature, forward LMP, and Henry Hub gas panels are climatology / forward-
curve representative shapes — clearly flagged "illustrative shape" in the
panel chrome.

### 5. New data pipeline file

`apps/gt-digital-twin/scripts/build_monthly_trajectories.py`

Reads `precomputed.json` and writes `client/public/monthly_trajectories.json`
(also copies to `dist/public/`). For each of the 12 grid cells it computes:

- monthly cumulative P10/P50/P90 envelopes for generation, spark margin, LTSA
  owner cost, net P&L, and fired hours (probability-weighted across paths)
- the empirical CDF of annual net P&L

plus three global monthly ensemble series (temperature, LMP, gas).

The seasonality weights and climatology profiles are inline-documented as
"illustrative" so it's clear which numbers are real engine output and which
are shape.

## Limitations

1. **Monthly shape is still illustrative for engine panels.** The engine emits
   annual totals only. Until `run_forward` writes per-month MWh / margin
   arrays, the cumulative curve shape for generation and gross margin is a
   weighted-monthly distribution of the real annual endpoint, not a real
   monthly trace. The panel-level "illustrative shape" badge calls this out.
2. **Temperature / LMP / gas panels are climatology**, not the engine's actual
   inputs for this run. The engine doesn't expose per-path monthly LMP /
   temperature / gas arrays through `precomputed.json` today. Marked
   "illustrative shape" with a per-panel caption.
3. **Bundle size warning** — main JS chunk is 769 KB unminified. Recharts is
   the heavy import. Code-split candidate for a v1.3 polish pass but not
   blocking for the showcase.
